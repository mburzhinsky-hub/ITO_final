# Stage 4 — расширение зон и связь с типами проектов

Выполнено требование по переходу от плоского списка зон к взаимосвязанной модели:

`Тип проекта → категории зон → шаблоны решений → состав сметы → проверки`.

## Что добавлено

1. Создана новая структура `src/data/zoneTaxonomy.js`:
   - 12 категорий зон;
   - 121 шаблон зоны;
   - карта `PROJECT_TYPE_ZONE_MODEL`;
   - helpers для получения категорий, шаблонов, default-наборов и seed-данных зоны.

2. Расширены типы проектов в `src/data/projectTypes.js`:
   - корпоративный офис;
   - музей / экспозиция;
   - конференц-центр;
   - retail / showroom;
   - образовательный объект;
   - диспетчерская / ситуационный центр;
   - HoReCa / отель;
   - спортивный объект;
   - event / сцена;
   - общественное пространство;
   - VR / симуляторы.

3. Обновлена страница `src/pages/ZonesPage.js`:
   - категории фильтруются по выбранному типу проекта;
   - внутри категорий показываются конкретные шаблоны зон;
   - добавлен режим «Показать все категории»;
   - нетиповые зоны добавляются без блокировки, но с предупреждением;
   - кнопка «Добавить типовые зоны» теперь использует `defaultZoneTemplateIds` типа проекта;
   - поиск учитывает название, категорию, назначение, AV-группы и теги;
   - карточки шаблонов показывают категорию, default-статус, инженерную проверку и причину рекомендации.

4. Обновлена модель зоны в `src/engine/projectFactory.js`:
   - `categoryId`;
   - `categoryName`;
   - `templateId`;
   - `requiredSystemGroups`;
   - `requiredDependencies`;
   - `typicalWorks`;
   - `requiresEngineerReview`;
   - `nonTypicalForProject`;
   - `recommendationReason`.

5. Обновлена карточка зоны `src/components/ZoneCard.js`:
   - бейдж категории;
   - статус нетиповой зоны;
   - причина рекомендации;
   - AV-группы;
   - зависимости;
   - инженерное предупреждение.

6. Обновлена генерация сметы `src/engine/estimate.js`:
   - учитывает `categoryId` и `templateId`;
   - приоритетно берёт `requiredSystemGroups` из шаблона зоны;
   - добавляет derived-строки по типовым работам: монтаж, ПНР, контент / ПО, логистика, конструкции, питание;
   - поддерживает генерацию по одной зоне через `zoneId`.

7. Обновлена проверка `src/engine/validation.js`:
   - проверяет наличие категории и шаблона у зоны;
   - предупреждает о нетиповой категории для выбранного типа проекта;
   - учитывает `requiresEngineerReview`;
   - сверяет обязательные AV-группы зоны с позициями сметы по этой зоне.

8. Добавлены стили Stage 4 в `src/styles.css`:
   - сетка категорий;
   - активные / нетиповые категории;
   - группировка шаблонов;
   - визуальное выделение нетиповых шаблонов и зон.

## Проверка готовности

- Синтаксис ключевых JS-файлов проверен через `node --check`.
- Импорт таксономии проверен: 12 категорий, 121 шаблон, 11 проектных моделей.
- Тестовая генерация сметы по зоне прошла успешно: зона создаётся из шаблона, смета получает derived-строки и позиции оборудования.
