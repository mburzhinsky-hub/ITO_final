# ВИЖУ AV пресейл-калькулятор

Статический многостраничный прототип без backend. Запускается как набор ES-модулей через локальный HTTP-сервер.

## Запуск

```bash
cd vizhu-architecture-stage1
python3 -m http.server 8080
```

Открыть: `http://localhost:8080`.

## Что сделано

- Hash-router: проекты, паспорт, зоны, смета, проверка, КП/экспорт, библиотека, настройки.
- Несколько локальных проектов через localStorage.
- Создание, открытие, дублирование, удаление проектов.
- Импорт и экспорт проекта в JSON.
- Единая модель Project.
- Каталог вынесен в `src/data/catalog.js`.
- Настройки вынесены в `src/data/defaultSettings.js` и `src/pages/SettingsPage.js`.
- Расчёт вынесен в `src/engine/pricing.js`, генерация сметы — в `src/engine/estimate.js`, проверка — в `src/engine/validation.js`.
- UI отделён от данных и расчёта.

## Ограничения этапа

- Расчётная математика специально не перепридумана глубоко.
- Экспорт Excel реализован как CSV-совместимая выгрузка, чтобы не зависеть от внешних библиотек.
- Полноценное редактирование каталога заложено архитектурно, но не реализовано.

## Stage 2: стабилизация расчётов

В этом архиве добавлен второй инженерный этап:

- единая функция `calculateProjectTotals(project, settings)` в `src/engine/pricing.js`;
- отдельные модули `currency.js`, `budget.js`, `estimate.js`, `validation.js`;
- разные формулы для `markup` и `margin`;
- контроль целевого бюджета с настройкой сравнения с ценой с НДС или без НДС;
- валютная модель `currency`, `unitCost`, `priceMode` (`manual`, `fixed`, `indexed`);
- предсказуемая генерация сметы с режимами `replace`, `append`, `recalculate-derived`;
- сохранение ручных строк при пересчёте derived-позиций;
- структурированные предупреждения с `severity`, `entityType`, `actionType` и сохранением принятых рисков;
- КП, CSV и JSON используют тот же расчётный слой, что и интерфейс.

## Stage 2 hotfix: библиотека оборудования

Исправлено отображение цен в библиотеке после перехода на валютную модель `currency/unitCost/priceMode`:

- нормализованные позиции каталога теперь сохраняют совместимое поле `priceRub`;
- страница библиотеки считает рублёвый эквивалент через `itemUnitCostRub(item, settings)`;
- для USD/indexed позиций показывается исходная цена в USD и текущий курс;
- добавление позиции из библиотеки продолжает использовать `unitCost`, `currency` и `priceMode`.


## Stage 3: UX/UI упрощение

В этом архиве добавлен третий продуктовый этап:

- линейный путь: Проекты → Паспорт → Зоны → Смета → Проверка → КП;
- новая пошаговая навигация и компактная сводка проекта;
- упрощённый паспорт проекта с раскрываемыми экспертными параметрами;
- отдельная полноценная страница зон;
- отдельная рабочая страница сметы с KPI, группировкой и режимами таблицы;
- отдельный этап проверки перед КП;
- понятные пустые состояния на ключевых экранах;
- библиотека и настройки оставлены в экспертной зоне;
- расчётное ядро и каталог поставщиков не переписывались.

Подробный отчёт: `STAGE3_UX_REPORT.md`.

## Stage 4 — зоны, категории и связь с типами проектов

Добавлен полноценный слой таксономии зон. Теперь калькулятор работает не с плоским списком, а с цепочкой:

`Тип проекта → категории зон → шаблоны решений → состав сметы → проверки`.

Ключевые файлы этапа:

- `src/data/zoneTaxonomy.js` — категории, шаблоны зон, связи с типами проектов, default-наборы;
- `src/data/projectTypes.js` — расширенные типы проектов с allowed / recommended / default zone ids;
- `src/pages/ZonesPage.js` — новый UX выбора зон по релевантным категориям;
- `src/components/ZoneCard.js` — отображение категории, AV-групп, зависимостей и инженерных предупреждений;
- `src/engine/estimate.js` — генерация сметы с учётом `categoryId` и `templateId`;
- `src/engine/validation.js` — проверки полноты AV-системы по зоне, категории и шаблону.

Подробности см. в `STAGE4_ZONE_MODEL_REPORT.md`.
