# ВИЖУ AV пресейл-калькулятор — архитектурный этап 1

Статический многостраничный прототип без backend. Запускается как набор ES-модулей через локальный HTTP-сервер.

## Запуск

```bash
cd vizhu-architecture-stage1
python3 -m http.server 8080
```

Открыть: `http://localhost:8080`.

## Что сделано

- Hash-router: проекты, паспорт, зоны, смета, проверка, КП/экспорт, библиотека, настройки.
- Несколько локальных проектов через localStorage.
- Создание, открытие, дублирование, удаление проектов.
- Импорт и экспорт проекта в JSON.
- Единая модель Project.
- Каталог вынесен в `src/data/catalog.js`.
- Настройки вынесены в `src/data/defaultSettings.js` и `src/pages/SettingsPage.js`.
- Расчёт вынесен в `src/engine/pricing.js`, генерация сметы — в `src/engine/estimate.js`, проверка — в `src/engine/validation.js`.
- UI отделён от данных и расчёта.

## Ограничения этапа

- Расчётная математика специально не перепридумана глубоко.
- Экспорт Excel реализован как CSV-совместимая выгрузка, чтобы не зависеть от внешних библиотек.
- Полноценное редактирование каталога заложено архитектурно, но не реализовано.

## Stage 2: стабилизация расчётов

В этом архиве добавлен второй инженерный этап:

- единая функция `calculateProjectTotals(project, settings)` в `src/engine/pricing.js`;
- отдельные модули `currency.js`, `budget.js`, `estimate.js`, `validation.js`;
- разные формулы для `markup` и `margin`;
- контроль целевого бюджета с настройкой сравнения с ценой с НДС или без НДС;
- валютная модель `currency`, `unitCost`, `priceMode` (`manual`, `fixed`, `indexed`);
- предсказуемая генерация сметы с режимами `replace`, `append`, `recalculate-derived`;
- сохранение ручных строк при пересчёте derived-позиций;
- структурированные предупреждения с `severity`, `entityType`, `actionType` и сохранением принятых рисков;
- КП, CSV и JSON используют тот же расчётный слой, что и интерфейс.
