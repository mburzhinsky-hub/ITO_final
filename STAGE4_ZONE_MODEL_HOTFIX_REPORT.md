# Stage 4 hotfix: зоны, оборудование и упрощение UX

## Что исправлено

1. Перепроверено покрытие всех шаблонов зон.
   - Всего шаблонов: 121.
   - Категорий: 12.
   - Дублей `templateId`: 0.
   - Пустых `recommendedItems`: 0.

2. Каждый шаблон теперь содержит заполненный состав оборудования.
   - Добавлено поле `recommendedItems` для каждого `ZoneTemplate`.
   - Для каждой позиции указаны `group`, `category`, `name`, `qty`, `priority`.
   - Состав формируется из обязательных AV-групп шаблона и нормализуется к реальным категориям каталога.

3. Исправлена нормализация AV-групп под фактический каталог.
   - `Камеры` → `PTZ-камеры`.
   - `Мониторы` → `LCD-панели`.
   - `AV-стойки` → `Доп. оборудование`.
   - `Питание` → `ИБП`.
   - `Видеостены` → `LED-экраны`.

4. Генерация сметы теперь опирается не только на категорию зоны, но и на заполненный состав `recommendedItems` конкретного шаблона.
   - Позиции выбираются из каталога по приоритетам шаблона.
   - Учитывается количество `qty` для типовых групп: микрофоны, акустика, LCD, LED, кабельная инфраструктура и т.д.
   - Лимит автоматического подбора увеличен до 12 позиций на зону, чтобы крупные зоны не теряли обязательные группы.

5. Проверка AV-системы теперь не ругается на корректно закрытые группы после генерации.
   - Прогнан тест по всем 121 шаблонам.
   - Для каждого шаблона создана зона, сгенерирована смета и запущена проверка.
   - Результат: 0 шаблонов без оборудования, 0 незакрытых обязательных AV-групп.

6. UX страницы «Зоны» упрощён.
   - Основной акцент: «Добавить типовые зоны», поиск и компактная библиотека.
   - Подробный состав оборудования скрыт в раскрывающийся блок.
   - Карточки шаблонов стали компактнее.
   - В карточке зоны подробные AV-группы и зависимости тоже спрятаны в раскрывающийся блок.
   - Интерфейс не перегружает пресейла, но экспертная информация остаётся доступной.

## Важное поведение после исправления

- Пользователь выбирает тип проекта.
- На странице зон видит релевантные категории и компактные шаблоны.
- При добавлении шаблона зона получает `categoryId`, `templateId`, `requiredSystemGroups`, `recommendedItems`, `requiredDependencies`, `typicalWorks`.
- Генератор сметы подбирает оборудование из библиотеки по `recommendedItems`.
- Проверка AV-системы валидирует закрытие обязательных групп уже по нормализованным категориям каталога.
