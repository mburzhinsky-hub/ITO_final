# Stage 5 · Библиотека оборудования, шаблоны и welcome-анимация

## Что изменено

### 1. Структурированная библиотека оборудования

Добавлен слой нормализации поверх текущих прайс-листов и базового каталога. Исходные поставщицкие позиции не удаляются физически, чтобы не потерять данные, но при построении `LIBRARY` теперь применяются:

- дедупликация по бренду, модели, артикулу и названию;
- нормализация валюты и цены;
- назначение `categoryId` и `subcategoryId`;
- назначение `solutionLevel` вместо плоского `scenario`;
- назначение `priceStatus`: `actual`, `estimated`, `unknown`;
- флаги `requiresPriceRequest`, `isPlaceholder`, `requiresEngineerReview`;
- привязки к `projectTypeIds`, `zoneCategoryIds`, `zoneTemplateIds`, `systemGroups`.

Основной каталог остаётся совместимым со старым кодом через поле `category`, но новые страницы и проверки используют структурные поля.

### 2. Категории оборудования

Создан файл:

- `/src/data/equipmentCategories.js`

В нём заведены корневые категории и подкатегории:

- Отображение;
- LED;
- ВКС;
- Звук;
- Коммутация и транспорт сигнала;
- Управление;
- Интерактив;
- Медиаплееры и вычисления;
- Кабели и расходники;
- Крепления и конструкции;
- Стойки и питание;
- Контент и ПО;
- Работы;
- Логистика и сервис.

Также добавлены быстрые фильтры для страницы библиотеки.

### 3. Зависимости

Созданы файлы:

- `/src/data/itemDependencies.js`
- `/src/engine/dependencyResolver.js`

Добавлены зависимости для дисплеев, проекторов, LED, ВКС, микрофонов, акустики и AV-стоек.

Зависимости не добавляются в смету принудительно. Они:

- показываются в карточке позиции;
- проверяются на странице «Проверка»;
- выводятся на странице «Библиотека»;
- могут быть добавлены как fallback-позиции вручную через кнопку.

### 4. Альтернативы

Созданы файлы:

- `/src/data/itemAlternatives.js`
- `/src/engine/alternativeResolver.js`

Система поддерживает явные альтернативы и автоматический подбор по категории / подкатегории:

- `budget`;
- `standard`;
- `premium`;
- `sameLevel`;
- `fasterDelivery`;
- `preferredSupplier`.

В карточке позиции отображается блок «Альтернативы». Если исходная позиция уже есть в смете, её можно заменить, при этом сохраняется история замены в строке сметы.

### 5. Уровни решений

У позиций теперь есть `solutionLevel`:

- `budget`;
- `standard`;
- `premium`;
- `expert`;
- `custom`.

Генерация сметы использует уровень проекта:

- для budget не тянет лишние premium/expert позиции;
- для premium отдаёт приоритет premium/expert и добавляет служебные premium-позиции: сервисный запас, расширенная ПНР, документация, проверка управления;
- если точного уровня нет, используется ближайший подходящий уровень.

На проверке показываются предупреждения, если строка сметы выше или ниже уровня проекта.

Файл `/src/data/installationTemplates.js` переписан в формат данных `InstallationTemplate`.

Добавлены группы:

- Переговорные;
- LED;
- ВКС;
- Музеи;
- Retail;
- Образование;
- Диспетчерские;
- HoReCa;
- Event;
- Инфраструктура.

Шаблоны связаны с:

- типами проектов;
- категориями зон;
- шаблонами зон;
- сценариями;
- сметными группами;
- зависимостями;
- типовыми работами;
- инженерными предупреждениями.

На странице библиотеки есть вкладка «Шаблоны инсталляций» и кнопка добавления шаблона в проект. Если конкретная позиция не найдена, создаётся fallback-позиция с `requiresEngineerReview` и `requiresPriceRequest`.

- карточка позиции с ценой, уровнем, статусом качества, зависимостями и альтернативами;
- добавление позиции в смету;
- добавление fallback-зависимостей;

### 8. Welcome-анимация

Созданы файлы:

- `/src/components/WelcomeIntro.js`
- `/src/styles/welcomeIntro.css`

Анимация встроена в `src/app/App.js` и подключена в `index.html`.

Поведение:

- показывается при первом открытии сайта в рамках сессии;
- использует `sessionStorage` с ключом `vizhuIntroSeen`;
- содержит текст «Добро пожаловать в мир ВИЖУ»;
- содержит кнопку «Пропустить»;
- автоматически исчезает примерно через 2.1 секунды;
- учитывает `prefers-reduced-motion`;
- не использует внешние библиотеки;
- не сбрасывает проекты и состояние калькулятора.

## Проверки

Выполнены статические проверки синтаксиса всех JS-файлов через `node --check`.

Дополнительно проверено, что:

- `LIBRARY` успешно строится из базового каталога и поставщицких прайсов;
- нормализованная библиотека содержит структурные поля;
- генерация сметы по зоне работает с новым уровнем решения;
- dependency resolver возвращает незакрытые зависимости для проверки.

## Важные замечания

1. Каталог физически не сокращался вручную, потому что он содержит десятки тысяч поставщицких позиций. Безопасная очистка реализована через нормализацию, дедупликацию и проверку качества при построении библиотеки.
2. Зависимости не добавляются принудительно при любом ручном действии. Пользователь видит их и сам решает, что добавить.
3. Полноценное редактирование всех полей карточки позиции пока не реализовано, но структура страницы подготовлена к добавлению CRUD-редактирования позже.
