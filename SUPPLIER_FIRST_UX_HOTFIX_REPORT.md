# Supplier-first / UX hotfix — задача 11

## Что изменено

- Добавлен компактный supplier-first индекс `src/data/supplierTemplateIndex.js`, собранный только из реальных файлов `public/data/suppliers/*.json`.
- Полный supplier-каталог по-прежнему не грузится на старте: большой прайс остаётся ленивым, а для шаблонных зон используется небольшой индекс предпочтительных позиций.
- Автогенерация сметы в `src/engine/estimate.js` теперь сначала ищет supplier-позиции, затем использует fallback из базовой библиотеки.
- В строки сметы добавлены source-метаданные: `sourceType`, `supplierName`, `supplierSku`, `supplierCategory`, `catalogRelevance`, `qualityVisibility`, `priorityScore`, `fallbackReason`, `templateRole`, `zoneTemplateId`, `replacementGroup`, `isPreferredForTemplates`.
- Карточки зон и таблица сметы получили понятные source-бейджи: supplier / fallback / manual / calculated.
- Проверка проекта показывает предупреждения, если для роли шаблона не найдена ни supplier-позиция, ни fallback.
- Добавлены supplier-first тесты и подключены к общему тест-раннеру.

## Как устроен подбор

1. Для каждой зоны берётся список рекомендуемых групп из шаблона зоны и связанных категорий.
2. Для каждой роли оборудования сначала проверяется supplier-first индекс.
3. Supplier-позиция допускается только если она:
   - AV-релевантная или инфраструктурная;
   - не скрыта по умолчанию;
   - не является спорной без расширенного режима;
   - не является IT/компьютерной вместо AV-роли;
   - имеет реальное название, поставщика и цену.
4. Если подходящей позиции поставщика нет, используется позиция базовой библиотеки с `sourceType: "baseLibrary"` и заполненным `fallbackReason`.
5. Если не найдено ничего, роль попадает в предупреждения проверки проекта, но расчёт не ломается.

## UX-упрощения

- Карточка зоны больше не показывает пользователю длинный каталог: сначала виден компактный рекомендуемый состав.
- Источники состава разделены на понятные статусы:
  - `Рекомендуется из прайсов`;
  - `Из базовой библиотеки`;
  - `Не найдено`.
- Детальная комплектация раскрывается внутри карточки зоны.
- В смете видно, какие строки пришли из supplier-прайсов, а какие являются fallback.
- Сложный supplier-каталог остаётся доступен через отдельный поиск/фильтры и не перегружает основной путь.

## Известные ограничения

- Supplier-first индекс — компактный слой для шаблонов, а не замена полному прайсу. Полный прайс остаётся в `public/data/suppliers` и подгружается лениво.
- Если поставщик указал слишком общую категорию или неполное описание, позиция может не попасть в compact preferred index и будет доступна только через расширенный каталог.
- Dependency-проверка может давать инженерные предупреждения по кабелям/креплениям/питанию, даже когда supplier-first состав уже создан. Это не ломает расчёт и требует отдельной настройки правил зависимостей при необходимости.
